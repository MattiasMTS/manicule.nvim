1. OVERVIEW                                                 *manicule-overview*

manicule.nvim provides per-project, extmark-anchored comments that are
decoupled from any particular buffer workflow (diffview, review tooling,
chat). Comments persist under `stdpath('state')/manicule/` as one file
per project root (mpack by default, JSON opt-in) and can be dispatched
to pluggable sinks (clipboard, PR drafts, webhooks, …).

Status: alpha — single-user, extmark-anchored, persisted under
`stdpath('state')/manicule/`. API will change.

2. SETUP                                                       *manicule-setup*

Full default configuration (every key is optional):

>lua
    require("manicule").setup({
      store = {
        -- Directory the per-root store lives under. Defaults to
        -- `stdpath('state').'/manicule/'` — out of the project tree.
        dir = vim.fn.stdpath("state") .. "/manicule/",
        -- On-disk envelope format. "mpack" (default) is binary and
        -- compact; "json" is human-readable.
        format = "mpack",
        -- When true, append the current git branch to the filename so
        -- notes are scoped per-branch. `main`/`master` collapse to the
        -- unsuffixed filename.
        branch = false,
        -- When true, buffers without a project root (no .git/.hg/
        -- package.json ancestor) are keyed by cwd instead of being
        -- ignored. Off by default.
        persist_unrooted = false,
        -- Markers passed to `vim.fs.root` to resolve the project root.
        root_markers = { ".git", ".hg", "package.json" },
      },
      ui = {
        width = 72,             -- floating editor width
        height = 6,             -- floating editor height
        editor_mode = "insert", -- "insert" or "normal"
        submit_keys = { "<CR>" },
        cancel_keys = { "q" },
        opacity = 0,            -- winblend (0 = opaque, 100 = clear)
        sticky = false,         -- true = always show comment popups;
                                -- false = only when the line is in view
      },
    })
<

                                                             *manicule-storage*

Store files live at >
    <store.dir>/<escaped-project-root>[%%<escaped-branch>].<format>
<
where `[\\/:]+` in each path segment is replaced with `%%`. Run
`:echo stdpath('state').'/manicule/'` to find the default directory.

                                                               *manicule-editor*

`:ManiculeAdd` and `:ManiculeEdit` open a floating markdown scratch
buffer. Type the body (multiple lines are allowed), then press `<CR>`
in normal mode to submit or `q` to cancel. Both key lists are
configurable via the `ui.submit_keys` / `ui.cancel_keys` arrays.

                                                              *manicule-popups*

Each comment is rendered as a per-record floating popup anchored to its
line. The popup title is the short record id (`c<6 chars>`) and the
footer shows the last-touched timestamp (`updated_at` or `created_at`)
followed by the keymap hint for the edit/delete `<Plug>` maps. Multiple
comments on the same line stack above each other in id order.

Set `ui.sticky = true` to keep every popup visible at all times; the
default (`false`) shows popups only for records whose line is currently
in the viewport, updating on scroll / resize / cursor movement.

                                                          *manicule-highlights*

manicule registers three highlight groups, all with `default = true` so
user overrides survive `:colorscheme` changes:

    ManiculeCommentBorder   Float border around comment popups.
    ManiculeCommentMeta     Popup title (`c<id>`) and footer hint.
    ManiculeLineNr          Line-number column tint on commented lines.
                            Default-linked to |hl-DiagnosticSignInfo|.

Override any of them with `:hi ManiculeLineNr ...` or
`vim.api.nvim_set_hl(0, "ManiculeLineNr", { ... })`.

                                                             *manicule-keymaps*

Default normal-mode keymaps (matching the popup footer hint):

    gca    Edit the comment at/covering the cursor.
    gcd    Delete the comment at/covering the cursor.

Set `vim.g.manicule_no_default_keymaps = 1` before the plugin loads to
opt out. All actions are also available as `<Plug>` maps:

    <Plug>(manicule-add)      n, x
    <Plug>(manicule-list)     n
    <Plug>(manicule-edit)     n
    <Plug>(manicule-delete)   n
    <Plug>(manicule-toggle)   n

                                                            *manicule-quickfix*

`:ManiculeList` populates the quickfix list (title prefixed with
`manicule`). The quickfix buffer gets two extra buffer-local normal-
mode bindings while the cursor sits on a manicule entry:

    dd     Delete the comment under the cursor.
    ce     Edit the comment under the cursor (opens the floating editor).
    <CR>   Native quickfix jump to the anchored location (unchanged).

The list refreshes in place while it stays open: `User
ManiculeAdded`, `ManiculeEdited`, `ManiculeDeleted`, `ManiculeResolved`,
and `ManiculeOrphaned` events all trigger a `setqflist` `"r"`-mode
replace that preserves the cursor row. A title-prefix check guards
against stomping on non-manicule quickfix lists (grep, diagnostics,
other plugins), so switching away mid-session is safe.

The `dd`/`ce` bindings honour the same opt-out flag as `gca`/`gcd`:
set `vim.g.manicule_no_default_keymaps = 1` to skip them.

3. COMMANDS                                                 *manicule-commands*

                                                                  *:ManiculeAdd*
:[range]ManiculeAdd         Create a comment anchored to [range] (or the
                            cursor line when no range is given).

                                                                 *:ManiculeList*
:ManiculeList               List all comments for the current project
                            (populates the quickfix list).

                                                                 *:ManiculeSend*
:ManiculeSend {sink}        Dispatch comments to {sink}. Completion lists
                            all registered sinks.

                                                              *:ManiculeResolve*
:ManiculeResolve [{n}]      Mark a comment as resolved. {n} is the 1-
                            indexed position in `:ManiculeList` order;
                            omit it to open a `vim.ui.select` picker.
                            Tab-completion returns valid positions
                            (`1`..`N`). Out-of-range or non-numeric
                            arguments error.

                                                               *:ManiculeDelete*
:ManiculeDelete [{n}]       Remove a comment and its anchor. {n} is a
                            1-indexed position in `:ManiculeList`
                            order; omit it to open the picker.

                                                                 *:ManiculeEdit*
:ManiculeEdit [{n}]         Prompt to replace a comment body. {n} is a
                            1-indexed position in `:ManiculeList`
                            order; omit it to open the picker.

                                                               *:ManiculeToggle*
                                                              *manicule-toggle*
:ManiculeToggle             Hide/restore all manicule visuals (popups +
                            line-number tints) without touching the
                            store. Records and anchor extmarks are
                            preserved, so orphan detection keeps
                            tracking edits while visuals are
                            suppressed. State is in-memory only and
                            resets on nvim restart. Fires a `User
                            ManiculeVisibility` autocmd on each flip
                            with `data.hidden = <bool>`. Also exposed
                            as `<Plug>(manicule-toggle)`.

                                                              *manicule-picker*

`:ManiculeEdit`, `:ManiculeDelete`, and `:ManiculeResolve` open a
`vim.ui.select` picker when invoked without an argument. Each row is
formatted as `<n> │ <path>:<line> │ <body…>` (resolved records show a
`[✓] ` prefix on the body). The picker and `:ManiculeList` share the
same record order (path → start line → id), so positional arguments
and picker selections are interchangeable.

The cursor-based `gca` / `gcd` keymaps are the fast at-cursor path;
the picker / positional commands are the "pick from a list" path.

4. PUBLIC API                                                    *manicule-api*

>lua
    local m = require("manicule")

    m.setup(opts)
    m.add({ range = {l1, l2}, body = "...", meta = {} })
    m.edit(id)
    m.delete(id)
    m.resolve(id)
    m.list({ path, unresolved, orphaned, author })
    m.send(sink_name, filter, ctx)
    m.register_sink(spec)
<

There is NO `m.on(event, fn)` helper. Subscribe via |nvim_create_autocmd|
on the `User` event — see |manicule-events|.

==============================================================================
5. SINKS                                                       *manicule-sinks*

A sink is any table matching: >lua

    {
      name     = "my-sink",
      send     = function(comments, ctx, cb) ... end,
      format   = function(comment) return "..." end, -- optional
      validate = function(ctx) return true end,      -- optional
    }
<

Register with: >lua

    require("manicule").register_sink(spec)
<

The bundled `clipboard` sink serves as a reference adapter.

==============================================================================
6. EVENTS                                                     *manicule-events*

Lifecycle events are fired as native |User| autocmds. Subscribe with
|nvim_create_autocmd|:

>lua
    vim.api.nvim_create_autocmd("User", {
      pattern = "ManiculeAdded",
      callback = function(ev)
        vim.print(ev.data) -- the newly-created record
      end,
    })
<

Patterns and payload shapes:

    ManiculeAdded          data = record
    ManiculeEdited         data = record (after update)
    ManiculeDeleted        data = { id, record }
    ManiculeResolved       data = record (resolved=true)
    ManiculeSent           data = { sink, count, ok, err }
    ManiculeOrphaned       data = { id, record }  (anchor invalidated on reload)
    ManiculeVisibility     data = { hidden }      (fired on :ManiculeToggle)
